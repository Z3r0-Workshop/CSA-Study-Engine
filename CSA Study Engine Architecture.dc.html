01

Ollama: two jobs, one client

Everything that touches the model goes through ollama_client.py, which knows only how to POST and return text. That single seam is what lets you swap Mistral for a fine-tuned model later without touching anything else.

JOB ① — generate
generator.py
Feed topic.name + blueprint_notes; ask for one CSA-style question as strict JSON (stem, options, answer, explanation). Reject & retry on malformed output.
JOB ② — grade
grader.py
For free-text answers: feed the question, the model answer, and the user's answer; ask for a 0–1 score plus a one-line rationale. Owns that prompt and nothing else.

03

The scheduler — the "smart" part

Spaced-repetition-lite, no SM-2. High exam weight + low recent accuracy = served first. Topics you've nailed fade out; unseen topics get a nudge so nothing stays untouched.

scheduler.py — next_topic_priority()
def next_topic_priority(topic):
    base   = topic.weight                  # exam importance
    recent = rolling_accuracy(topic, n=10) # 0..1, def 0.5
    unseen = 0.3 if attempts(topic) == 0 else 0
    return base * (1 - recent) + unseen
Weight × miss-rate. Important topics you keep missing rise to the top.
Mastery fades. High rolling accuracy drives priority toward zero.
Nothing unseen. A +0.3 bonus guarantees fresh topics surface early.

Why this stack

SQLite
Single-user study tool. Postgres here would be resume-padding — reviewers notice. Zero setup, one file, ships in the image.
FastAPI
A real, documented API surface — the auto-generated /docs is a great demo screenshot — while still runnable as a plain script.
Ollama
Already running locally — keeps the project zero-cost and offline. A genuine architectural point, not a limitation.
